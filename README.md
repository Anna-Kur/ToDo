# Файлы для итогового задания

В директории `tests` находятся тесты для проверки API, которое должно быть реализовано в веб-сервере.

Директория `web` содержит файлы фронтенда.

Директория `pkg` содержит файлы бекэнда.

*выполнены все задания повышенной сложности

Веб-приложение для управления задачами с поддержкой повторяющихся событий. Позволяет создавать, редактировать, удалять задачи и отмечать их выполненными. Поддерживает гибкие правила повторения (ежедневно, еженедельно, ежемесячно, ежегодно).

# Локальный запуск

Порт сервера (по умолчанию 7540)
export TODO_PORT=7540

Путь к файлу базы данных (по умолчанию scheduler.db)
export TODO_DBFILE=./scheduler.db

Пароль для аутентификации (опционально)
export TODO_PASSWORD=password

Скачать зависимости
go mod download

Запустить сервер
go run main.go

После запуска сервер будет доступен по адресу:
http://localhost:7540

# Пример командной строки для запуска контейнера:

 docker build -t scheduler-app .
 docker run -d \
 --name scheduler \
 -p 7540:7540 \
 -v "$(pwd)/data:/app/data" \
 -e TODO_PASSWORD=password \
 scheduler-app 

# Запуск тестов

В tests/settings.go у переменных FullNextDate и Search должно быть значение true - для тестирования заданий повышенной сложности

Для тестирования аутентификации необходимо:
1. Запуститm сервер с паролем
export TODO_PASSWORD=password
go run main.go &

2. Получите токен
curl -X POST http://localhost:7540/api/signin \
  -H "Content-Type: application/json" \
  -d '{"password":"mysecret"}'

3. Скопируйте токен в tests/settings.go в переменную Token
